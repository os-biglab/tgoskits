# `host-axsched-dedicated` 维护者说明

本文件只供出题人、评审和比赛平台维护者使用，不对选手公开。

## 1. 出题意图

- 目标组件/模块：`axsched`
- 主要路径：
- `components/axsched`
- 试运行目标：验证单组件专属题在预赛中的区分度。

## 2. 标准答案摘要

- 真实 bug 位置：按正式出题时补充。
- 根因类别候选：`runqueue 不变量破坏`, `时间片更新错误`, `饥饿与优先级反转`
- 期望修复方向：保持最小补丁、补回归测试、避免样例定制。

## 3. visible tests 设计意图

- 当前 visible commands：
- `cargo test -p axsched`
- 它们应帮助选手进入问题现场，但不应直接暴露 hidden tests 的输入空间。

## 4. hidden tests 矩阵

| 类别 | 覆盖点 | 预期拦截的投机策略 |
| --- | --- | --- |
| `hidden-scheduler` | 补不同调度顺序、时间片和优先级组合 | 只修公开样例、删除分支、静默吞错 |
| `hidden-regression` | 验证 runqueue 相关路径没有被修坏 | 只修公开样例、删除分支、静默吞错 |
| `hidden-stability` | 验证修复不会引入新的饥饿或不公平行为 | 只修公开样例、删除分支、静默吞错 |

## 5. 人工复核要点

- 是否只修改了允许路径
- 是否补了回归测试或等价复现脚本
- 根因分析是否与补丁一致
- 是否具备回流到正式仓库的潜力
